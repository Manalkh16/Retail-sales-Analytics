# Data Folder

## Raw Dataset

The raw dataset (`online_retail_II.xlsx`) is **not committed to this repository** due to file size (~45MB).

**Download it directly from the UCI ML Repository:**

🔗 **[Download online_retail_II.xlsx](https://archive.ics.uci.edu/dataset/502/online+retail+ii)**

After downloading, place the file in this folder:

```
Retail-Analytics-Project/
└── data/
    └── online_retail_II.xlsx   ← place it here
```

Then run the cleaning pipeline:

```bash
python notebooks/01_data_cleaning.py
```

This generates three derived files automatically:

| File | Description | Generated by |
|------|-------------|--------------|
| `online_retail_clean.csv` | 824K cleaned transactions + 10 engineered features | `01_data_cleaning.py` |
| `customer_summary.csv` | 5,878 customers with RFM scores and segment labels | `01_data_cleaning.py` |
| `customer_clusters.csv` | Customer summary + K-Means cluster assignments | `03_advanced_analysis.py` |

## Dataset Overview

| Property | Value |
|----------|-------|
| Source | UCI Machine Learning Repository |
| Name | Online Retail II |
| Period | December 2009 – December 2011 |
| Raw rows | 1,067,371 invoice lines |
| Clean rows | ~824,000 |
| Countries | 38 |
| Unique customers | 5,878 |
| Unique SKUs | ~4,000 |

## Column Reference

| Column | Type | Description |
|--------|------|-------------|
| Invoice | VARCHAR | Unique order identifier (prefix 'C' = cancellation) |
| StockCode | VARCHAR | Product / SKU code |
| Description | VARCHAR | Product name |
| Quantity | INT | Units sold per line item |
| InvoiceDate | TIMESTAMP | Date and time of order |
| Price | DECIMAL | Unit price in GBP (£) |
| Customer ID | VARCHAR | Unique customer identifier |
| Country | VARCHAR | Customer's country |

## Engineered Features (added by 01_data_cleaning.py)

| Feature | Description |
|---------|-------------|
| `revenue` | `quantity × price` — line-level revenue |
| `year`, `month`, `quarter` | Temporal decomposition |
| `month_name` | Human-readable month label |
| `day_of_week` | 0=Monday … 6=Sunday |
| `is_weekend` | Boolean — Saturday or Sunday |
| `hour` | Hour of order (0–23) |
| `country_clean` | Normalised country name |
| `order_id` | Alias for invoice |
